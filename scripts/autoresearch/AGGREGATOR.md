# AutoCast Aggregator (Phase 3.4)

Der Aggregator aggregiert alle Ergebnisse eines AutoResearch Runs und generiert den `CYCLE_REPORT.md`.

## Verwendung

### CLI

```bash
# Basis-Usage
node aggregator.js --runDir reports/autoresearch/runs/20260325_021955

# Mit Baseline-Score
node aggregator.js --runDir reports/autoresearch/runs/20260325_021955 --baseline 0.2670

# Mit Custom Output-Pfad
node aggregator.js --runDir <path> --output /custom/path/REPORT.md

# Zusätzlich JSON-Output
node aggregator.js --runDir <path> --json
```

### Als Module

```javascript
import {
    aggregateRun,
    generateCycleReport,
    updateStatusWithSummary,
    getScoreHistory
} from './aggregator.js';

// Aggregiere einen kompletten Run
const results = aggregateRun('./runs/20260325_021955');

// Generiere Markdown-Report
const report = generateCycleReport(results, 0.2670); // mit Baseline

// Update STATUS.json
updateStatusWithSummary('./runs/20260325_021955', results.summary, {
    overallStatus: 'COMPLETED',
    finalObjectiveScore: 0.3123
});

// Score-Verlauf extrahieren
const history = getScoreHistory(results.jobs);
```

## API-Referenz

### `aggregateRun(runDir)`

Aggregiert alle Ergebnisse eines Runs.

**Parameter:**
- `runDir` (string): Pfad zum Run-Verzeichnis (enthält STATUS.json)

**Rückgabe:** `object|null`
```javascript
{
    runId: "20260325_021955",
    timestamp: "2026-03-25T02:15:18.003Z",
    schemaVersion: "1.4.0",
    summary: {
        totalJobs: 9,
        keepCount: 3,
        rejectCount: 5,
        failedCount: 1,
        pendingCount: 0,
        runningCount: 0,
        unknownCount: 0
    },
    jobs: [/* Job Details */],
    scoreHistory: [/* Score Steps */],
    status: {
        overallStatus: "COMPLETED",
        completedAt: "2026-03-25T02:45:12.123Z"
    }
}
```

### `generateCycleReport(results, baselineScore?)`

Generiert den Markdown-Report.

**Parameter:**
- `results` (object): Ergebnis von `aggregateRun()`
- `baselineScore` (number, optional): Baseline-Score (default: erster Job-Before-Wert)

**Rückgabe:** `string` - Markdown-Content

### `updateStatusWithSummary(runDir, summary, options?)`

Updated STATUS.json mit der Zusammenfassung.

**Parameter:**
- `runDir` (string): Pfad zum Run-Verzeichnis
- `summary` (object): Summary-Objekt
- `options` (object, optional): Zusätzliche Optionen
  - `overallStatus` (string): Gesamt-Status
  - `finalObjectiveScore` (number): Finaler Score
  - `reportPath` (string): Pfad zum Report

**Rückgabe:** `boolean` - Erfolg

### `getScoreHistory(jobs)`

Extrahiert den Score-Verlauf aus Job-Ergebnissen.

**Parameter:**
- `jobs` (array): Liste der Job-Details

**Rückgabe:** `array`
```javascript
[
    { step: "baseline", methodId: "initial", score: 0.2670 },
    { step: "silence_overlap_bleed_weight", methodId: "silence_overlap_bleed_weight", decision: "KEEP", score: 0.2890, change: 0.0220 },
    { step: "duration_padding_rebalance", methodId: "duration_padding_rebalance", decision: "KEEP", score: 0.3012, change: 0.0122 }
]
```

### `extractMetrics(result)`

Extrahiert Metriken aus einem Result-Objekt.

**Parameter:**
- `result` (object): Result-Objekt aus STATUS.json

**Rückgabe:** `object`
```javascript
{
    objectiveScore: 0.2890,
    speechRecall: 0.5933,
    reviewRecall: 0.0000,
    ignoreRecall: 0.0000,
    durationGoodOrNearRatio: 0.0000,
    decision: "KEEP",
    improvement: 0.0220
}
```

### `calculateImprovement(baseline, final)`

Berechnet die prozentuale Verbesserung.

**Parameter:**
- `baseline` (number): Baseline-Score
- `final` (number): Finaler Score

**Rückgabe:** `string` - Formatierte Verbesserung (z.B. "+16.97%" oder "-5.23%")

## Customization

### Custom Report Template

Du kannst den Report-Generator erweitern:

```javascript
import { generateCycleReport } from './aggregator.js';

// Generiere Basis-Report
let report = generateCycleReport(results, baseline);

// Füge eigenen Abschnitt hinzu
report += `\n\n## Custom Analysis\n\n`;
report += `Meine eigene Analyse: ...\n`;

writeFileSync('CUSTOM_REPORT.md', report);
```

### Custom Status-Update

```javascript
import { updateStatusWithSummary } from './aggregator.js';

updateStatusWithSummary(runDir, summary, {
    overallStatus: 'COMPLETED',
    finalObjectiveScore: 0.3123,
    // Custom Fields
    customMetrics: {
        topPerformer: 'silence_overlap_bleed_weight',
        totalImprovement: 0.0453
    },
    analysis: {
        recommendation: 'Deploy changes from KEEP methods',
        risk: 'LOW'
    }
});
```

### Score-History Customization

```javascript
import { getScoreHistory } from './aggregator.js';

const history = getScoreHistory(jobs);

// Nur KEEP-Einträge
const keepHistory = history.filter(h => h.decision === 'KEEP');

// Mit Custom-Transformation
const transformed = history.map(h => ({
    ...h,
    percentage: h.change ? (h.change * 100).toFixed(2) + '%' : 'N/A'
}));
```

## Integration

### Mit Dispatch Processor

```javascript
// Nach dem Dispatch
const dispatch = await dispatchMethods(plan);

// Warte auf Completion (Polling)
const completed = await pollUntilComplete(runDir);

// Aggregiere Ergebnisse
if (completed) {
    const results = aggregateRun(runDir);
    const report = generateCycleReport(results, baselineScore);
    // ...
}
```

### Mit Status Manager

```javascript
import statusManager from './lib/status_manager.js';
import { aggregateRun } from './aggregator.js';

const status = statusManager.loadStatus(runDir);

if (statusManager.isComplete(status)) {
    const results = aggregateRun(runDir);
    updateStatusWithSummary(runDir, results.summary, {
        overallStatus: 'COMPLETED'
    });
}
```

```
STATUS.json
    │
    ├── Job 1 (method1)

    │     ├─ result.decision: "REJECT"
    │     └─ ...
    │
    └── Job N (methodN)
          └─ ...

              │
              ▼

      aggregateRun(runDir)
              │
              ├── Zähle Jobs nach Status
              ├── Extrahiere Score-History
              └── Sammle Job-Details

              │
              ▼

      generateCycleReport(results)
              │
              └── CYCLE_REPORT.md

              │
              ▼

      updateStatusWithSummary(runDir, summary)
              │
              └── STATUS.json (mit summary + finalScore)
```

## Fehlerbehandlung

### Common Issues

**STATUS.json nicht gefunden:**
```
❌ STATUS.json nicht gefunden: /path/to/run/STATUS.json
```
→ Prüfe den `--runDir` Pfad

**Leere oder ungültige STATUS.json:**
```
❌ Fehler beim Laden von STATUS.json: Unexpected token...
```
→ Validiere das JSON-Format

**Keine Jobs gefunden:**
→ Report wird generiert, aber Summary zeigt 0 Jobs
